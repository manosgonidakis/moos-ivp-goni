# Skills

Canonical, agent-neutral skill folders belong here.

Each skill should contain:

- `SKILL.md`
- optional `references/`
- optional `scripts/`
- optional `assets/`
- optional product-specific UI metadata under `agents/`

Populated skill directories are intended to validate with the local
`skill-creator` quick validator. Experimental future skills should not be added
as empty placeholder directories; keep them in `docs/` until they have a
`SKILL.md`.
